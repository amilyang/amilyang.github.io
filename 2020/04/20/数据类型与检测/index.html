数据类型与检测

知识图谱

JavaScript 中的数据类型

数据类型包括基本数据类型和引用数据类型

  • 基本数据类型： String，Number、Boolean、Null、Undefined、Symbol
  • 引用数据类型： Object、Function、Array、Date、 RegExp

数据类型与堆栈

堆和栈都是运行时内存中分配的一个数据区，因此也被称为堆区和栈区； 二者存储的数据类型和处理速度不同：

  • 堆（heap）用于复杂的数据类型（引用类型）分配空间， 例如数组对象、Object对象；它是运行时动态分配内存的，因此存取速度较慢。
  • 栈 （stack）中主要存放一些基本类型的变量和对象的引用（指针），其优势是存取速度比堆要快，并且栈内的数据可以共享，但是缺点是存在栈中的数据大小和生存期必须是确定的，缺乏灵活性

基本数据类型存储在栈内存中，一个变量复制时，会在栈中分配一个新的位置，并把变量的值存放在新的位置上，新变量的值发生改变时不会影响到被复制的变量的值

引用类型将数据存在堆内存（heap）中，而变量作为指针存放在栈内存（stack）中，当我们访问引用数据时，先从栈内存（stack）中取出指针，通过指针（索引）在堆（heap）内存中，通过指针在堆内存（heap）中找到对应的数据。

数据类型的检测方式

  • typeOf
  • instanceOf
  • constructor
  • Object.prototype.toString.call()

typeof

console.log(typeof 123) //number
console.log(typeof '123') //string
console.log(typeof true) //boolean
console.log(typeof undefined) //undefined
console.log(typeof null) //object
console.log(typeof NaN) //number
console.log(typeof []) //object
console.log(typeof {}) //object
console.log(typeof function() {}) //function
console.log(typeof new Date) //object

注意：
typeof 操作符只可以对null以外的基本类型作出正确的判断，对于引用数据类型，检测出来的都是Object，无法判断具体属于哪种对象

instanceof

instanceof是用来判断A是否为B的实例，表达式为 A instanceof B，如果A是B的实例，返回true，否则返回false

console.log("1" instanceof String) //false
console.log(1 instanceof Number) //false
console.log(true instanceof Boolean) //false
console.log([] instanceof Array) //true
console.log([] instanceof Object) //true
console.log(function(){} instanceof Object) //true
console.log({} instanceof Object) //true

注意：

  • instanceof 只能用来判断两个对象是否属于实例关系，而不能判断对象实例具体属于哪种类型
  • instanceof 无法对原始类型进行判断，所以一般都是在typeof判断为object时才使用instanceof
  • instanceof可以准确的判断复杂数据类型，但是不能正确判断基本数据类型

instanceof原理

instanceof是通过原型链判断的， A instanceof B，在A的原型链中层层查找，是否有原型等于B.prototype
如果一直找到A的原型链顶端（null，即Object.prototype.proto）仍然不能等于B.prototype,那么返回false，
否则返回true

代码实现
function _instanceof(A, B){ // A表示左表达式， B表示右表达式
  var L = A.__proto__  // 取A的隐式表达式
  var R = B.prototype  // 取B的显示表达式
  while(true){  
    if(L === null) //已经找到顶层
      return false
    if(L = R){  // 当R严格等于L时，返回true
      return true
    } else {  
      L = L.__proto__ // 继续向上一层原型链查找
    }
  }
}

  • 补习一下基础支持 ： break 退出当前循环； continue 退出单次循环，执行下一次循环； return 退出当前方法，不往下执行，返回该方法调用处往下执行

  • 新增知识点：ES6 当中 Object.getPrototypeOf()可以用来从子类上获取父类

Object.getPrototypeOf(children) === parent

因此可以使用这个方法判断一个类是否继承了另一个类

construtor

construtor 可以打印出实例所属的类， 用法： （实例).construtor

console.log(('abc').constructor === String) //true
console.log((1).constructor === Number) //true
console.log((true).constructor === Boolean) //true
console.log(([]).constructor === Array) //true
console.log(([]).constructor === Object) //false
console.log(({}).constructor === Object) //true
console.log((function(){}).constructor === Function) //true
console.log((function(){}).constructor === Object) //false

注意：

  • null 、undefined因为没有 constructor存在，因此这两个不能通过这种方式来判断
  • 如果重写对象的prototype， 原有的constructor会丢失， 判断结果会发生改变

Object.prototype.toString.call()

首先了解一下为什么用Object.prototype.toString()而不是obj.toString() ?
因为toString是Object的原型方法，而Array、Function等类型作为Object的实例，都重写了toString方法。
不同的对象类型调用toString方法时，根据原型链的知识，调用的是对应的重写之后的toString方法（Function 类型返回内容为函数体的字符串， Array类型返回元素组成的字符串……）而不会去调用Object原型上的toString方法（返回对象具体类型），所以采用obj.toString()不能得到其对象类型，只能将obj转换为字符串类型；因此，在想要得到对象的具体类型时，应该调用Object原型上的toString方法

  • 验证

Object是js中所有其他数据类型的父类。意思是所有的数据类型都继承了Object， 调用Object原型上的toString方法会默认返回当前对象的[[Class]]属性的值，[[Class]]是一个内部属性，所有对象（原生和宿主对象）都拥有该属性，其格式为[object Xxx], 如数组：[object Array]

console.log(Object.prototype.toString.call('abc'))          // "[object String]"
console.log(Object.prototype.toString.call(123))            // "[object Number]"
console.log(Object.prototype.toString.call(true))           // "[object Boolean]"
console.log(Object.prototype.toString.call(null))           // "[object Null]"
console.log(Object.prototype.toString.call(undefined))      // "[object Undefined]"
console.log(Object.prototype.toString.call([]))             // "[object Array]"
console.log(Object.prototype.toString.call({}))             // "[object Object]"
console.log(Object.prototype.toString.call(function(){}))   // "[object Function]"
console.log({}.toString()) //[object Object]

注意：

  • object直接调用toStringf（）就能返回[object Object], 但是其他对象需要通过call/apply才能返回正确的类型信息
  • 我们也发现通过Object.prototype.toString.call()可以准确的检测js的所有数据类型
