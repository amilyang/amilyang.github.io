new 关键字详解

new 运算符是用来实例化一个类，从而在内存中分配一个实力对象。 那为什么要通过new来产生对象呢？

例子

代码解读

1-3行 创建了函数Person，并为它定义了属性name， name的值是函数被执行时的形参。
第4行 在Person对象（Person本身也是一个函数对象）上定义了一个静态属性：age，并赋值 18
5-7行 在Person函数的原型对象上prototype上定义了一个getName()方法, getName方法输出了this的name值
第9行 通过new关键字创建了一个新对象student
10-14行 student对象尝试访问name和age属性，并调用getName方法
16-20行 Person对象尝试访问name和color属性

重点解析

第9行代码是关键：

JS引擎执行这句话时，在内部做了很多工作，下面大概模拟一下工作流程：

  1. 创建空对象obj
  2. 把obj的 obj 指向Person的原型对象prototyoe, 此时便建立了obj对象的原型链：obj->Person.prototype->Object.prototype->Null
    [可参阅 JS原型和原型链]
  3. 在obj对象的执行环境调用Person函数并传递参数“xiaoming”， 相当于 var result = obj.Person(‘xiaoming’).
    这句话执行完之后，obj便产生了属性name并赋值为“xiaoming”
  4. 判断第3步的返回值，如果无返回值或返回值不是一个对象，则将obj作为新对象返回， 否则将返回值作为新对象返回。

理解new的运行机制后，我们知道student其实就是过程4的返回值，因此我们对student对象的认知就多了一些

  • student的原型l链是 student ->Person.prototype->Object.prototype->null *

  • student上新增了一个属性name*

    分析完了student的产生过程，我们再看看输出结果：
    student.name -> 再过程3中，obj产生了name属性。 student.name 就是这个理的obj.name
    sutdent.age -> student回先查找自身的age属性，没有找到便会沿着原型链查找，在上述例子中，我们仅在Proson对象上定义了age，并没有在其他原型链上定义，因此找不到。
    student.getName ->studen会现在自身查找getName方法，没找到会沿着原型链查找，在上述的例子中，我们在Person的原型链上定义了getName，因此在原型链上找到了getName

    另外getName方法还访问了this.name, 这里的this指的是其他调用者obj，因此输出的是obj.name的值

    对于Person来说，它本身也是一个对象，因此，它在访问属性和方法时也遵守上述查找规则所以：
    Person.age -> 18
    Person.name -> “Person”, Person先查找自身的name，找到了name，
    注意：这个name不是我们定义的name，而是函数对象内置的属性。

    一半情况下，函数对象在产生时会内置name属性并将函数名作为赋值（仅函数对象）

    Person.getName -> 在自身没找到getName方法，也会沿着原型链查找，那Person的原型链是什么呢？
    Person->Function.prototype->Object.prototype-null

    因此Person的原型链上没有定义getName方法！

    new 存在的意义

    通过上面的分析，我们发现了student继承了Person的部分属性，因此我们可以简单的理解：Person和student是继承关系。
    另一方面student是通过new产生的对象，那么student到底是不是Person的实例对象呢？
    我们来了解一下JS是如何来定义“实例对象”的

    a instanceof b

如果为true，就证明A是B的实例对象，我们来判断一下student和Person

总结

在javascript中，通过new可以产生原对象的一个实例对象，而这个实例对象继承了原对象的属性和方法。因此。new存在的意义在于它实现了js中的继承（inherit），而不仅仅是实例化了一个对象
